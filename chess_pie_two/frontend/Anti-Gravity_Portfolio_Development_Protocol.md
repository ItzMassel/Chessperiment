# Anti-Gravity Portfolio Development Protocol

## Overview
Custom skill for collaborative front-end portfolio development with Lasse. Follows a structured, iterative process with checkpoints at each major phase.

## Phase 1: Discovery & Planning

### Step 1: Sitemap Creation
- **Action**: Draft complete sitemap based on project context
- **Deliverable**: Present sitemap to Lasse for feedback
- **Next**: Incorporate feedback and finalize site structure

### Step 2: Wireframe Development  
- **Action**: Create 3+ wireframe variants (V1, V2, V3)
- **Deliverable**: Visual mockups showing different layout approaches
- **Next**: Lasse chooses preferred wireframe, provide feedback

### Step 3: Design Assets
- **Required from Lasse**: 
  - Color palette
  - Font selections  
  - Real content (not placeholders)
  - NotebookLM context document

## Phase 2: Development Protocol

### Technical Stack
- **Framework**: Next.js with maximum server components
- **Styling**: Normal CSS
- **Animations**: Framer-motion
- **Approach**: Desktop-first, responsive from start

### Development Process
1. **Build ONE complete page first** (homepage priority)
2. **Per section**: Apply styling, fonts, colors, layout
3. **Exclusions**: No interactions/animations yet
4. **Content**: Use real content (no placeholders)
5. **Review**: Present full page to Lasse

### Feedback Loop
- **Question**: "What specifically should I change?"
- **Approach**: Single iteration, no multiple code variants
- **Documentation**: Remember preferences (white space, etc.)

## Phase 3: Scale & Completion

### Page Development
1. Apply all homepage learnings to remaining pages
2. Build all other pages simultaneously  
3. Maintain consistency with established patterns

### Final Polish
1. Add all interactions and animations
2. Complete functionality implementation
3. Final review and adjustments

## Key Principles

### Communication Style
- Ask specific questions about changes needed
- Present options during planning phases only
- No assumptions - get explicit approval

### Memory & Consistency
- Track Lasse's preferences across sessions
- Apply learned patterns to all subsequent pages
- Maintain white space and design consistency

### Quality Checkpoints
- Sitemap approval before wireframes
- Wireframe selection before development  
- Homepage perfection before scaling
- Final review of complete site

## Success Criteria
- Portfolio reflects Lasse's vision at every step
- Consistent design patterns across all pages
- Optimized for performance with server components
- Smooth animations and interactions
- Fully responsive design
